# Звіт з лабораторної роботи: Контролер рою дронів

## Аналіз Завдання 1 (I/O Bound - Потоки)
У першому завданні ми моделювали операції дрона (політ, зчитування, завантаження), які здебільшого є **I/O-bound** (очікування введення-виведення / мережі). Функція `time.sleep()` ідеально імітує затримки на мережу.
*   **Чому потоки працюють:** Завдяки Global Interpreter Lock (GIL) в Python лише один потік може виконувати байт-код одночасно. Однак, під час виклику I/O операцій (як `time.sleep` або мережеві запити), GIL звільняється, дозволяючи іншим потокам працювати.
*   **Результати:** Багатопоточна версія продемонструвала значне прискорення (Speedup) у порівнянні з послідовною. Час виконання скоротився майже пропорційно кількості потоків, за винятком вузького місця — семафора на завантаження (обмеження в 3 канали).

## Аналіз Завдання 2 (CPU Bound - Процеси)
У другому завданні дрони виконують "важкі обчислення". Це класична **CPU-bound** задача (навантаження на процесор).
*   **Чому тут потрібні процеси, а не потоки:** Якби ми використовували `threading` для математичних обчислень у Python, GIL блокував би виконання інших потоків. Програма працювала б так само повільно (або навіть повільніше через накладні витрати на перемикання контексту), як і послідовна версія.
*   **Як допоміг `multiprocessing`:** Модуль `multiprocessing` створює окремі процеси ОС, кожен з яких має власний інтерпретатор Python та свій власний GIL. Це дозволило використати всі доступні ядра процесора.
*   **Семафори та черги:** `mp.Semaphore` дозволив нам не перевантажити систему (обмежили кількість воркерів), а `mp.Queue` гарантувала безпечну передачу даних від дочірніх процесів до головного.

## Висновок
1.  **Threading** ефективний для задач з інтенсивним очікуванням (I/O, мережа, запити до баз даних).
2.  **Multiprocessing** необхідний для паралелізації важких математичних або аналітичних розрахунків у Python для обходу обмежень GIL.
3.  Архітектура інтеграції (Producer-Consumer з Завдання 3) є найбільш оптимальним шаблоном для сучасних систем: потоки збирають I/O дані і передають їх у чергу, з якої процеси паралельно виконують обчислення.
